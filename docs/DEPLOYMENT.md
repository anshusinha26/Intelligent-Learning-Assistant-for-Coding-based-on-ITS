# Deployment

## Local Deployment (Recommended for Development)

### Backend

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 scripts/manage_migrations.py upgrade --db-path data/coding_assistant.db
python3 load_sample_data.py
PORT=8020 python3 -m src.main
```

### Frontend

```bash
cd frontend
npm install
VITE_API_URL=http://localhost:8020/api npm run dev -- --host 0.0.0.0 --port 5173
```

## Container Deployment

### Development profile

```bash
docker compose --profile dev up --build
```

### Production profile

```bash
docker compose --profile prod up --build
```

Production profile includes:

- backend service
- frontend service
- prometheus
- grafana
- health checks and startup ordering

## Environment Setup

1. Copy a template:
   - `cp .env.example .env`
2. Set production secrets and policies:
   - `SECRET_KEY`
   - strict password policy variables
   - explicit `CORS_ALLOW_ORIGINS`
   - `HSTS_ENABLED=true`
3. Set frontend API URL:
   - `VITE_API_URL=https://<backend-domain>/api`

## Post-Deployment Verification

- API health: `GET /api/health`
- Readiness: `GET /api/readiness`
- Metrics: `GET /api/metrics`
- Auth login flow
- Problems endpoint returns premium-only records
- Dashboard and recommendations load for authenticated user

## Render Deployment (Backend)

This repository now includes `render.yaml` for backend deployment.

### 1. Create Service

1. In Render dashboard, choose **New +** -> **Blueprint**.
2. Connect this repository and select branch.
3. Render reads `render.yaml` and creates service `ila-coding-backend`.

### 2. Configure Required Secrets

In Render service -> **Environment**, set:

- `CORS_ALLOW_ORIGINS=https://<your-vercel-domain>`

`SECRET_KEY` is auto-generated by the blueprint (`generateValue: true`).

### 3. Storage on Free Plan

The free Render plan uses ephemeral filesystem storage. This blueprint sets:

- `DB_PATH=data/coding_assistant.db`

Implication: SQLite data is reset if the instance is recreated or a fresh deploy environment is provisioned.
For capstone/demo usage this is acceptable.
For persistent production data, move to a paid Render plan (persistent disk) or managed Postgres.

### 4. One-Time Premium Data Load

After first deploy, open Render shell and run:

```bash
python load_sample_data.py
```

This loads premium problems into the runtime DB (legacy loading stays disabled by default).

### 5. Verify Backend

Replace `<render-backend-url>` with your service URL:

```bash
curl -s https://<render-backend-url>/api/health
curl -s https://<render-backend-url>/api/liveness
curl -s https://<render-backend-url>/api/readiness
curl -s -o /dev/null -w "%{http_code}\n" https://<render-backend-url>/docs
curl -s -o /dev/null -w "%{http_code}\n" https://<render-backend-url>/openapi.json
```
